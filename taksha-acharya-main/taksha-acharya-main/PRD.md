# Taksha Acharya v5 — Product Requirements Document

**For:** Product Engineer building in Stitch / AI Studio
**Date:** 2026-04-08
**Status:** Phase 1 complete (Next.js + Supabase). This PRD covers the full product.

---

## 1. What Is This

A mobile-first training app for Indian carpentry trainees certified under Vocational Qualification Pack AGR/Q0405 (Carpentry Trainee, NSQF Level 4). The app teaches 38 modules across 390-540 hours of curriculum in Bengali, Hindi, and English — with AI chat, voice, quizzes, and video.

The persona is **Taksha** — a warm, patient, senior Kaarigar (field worker) who trains new growers. He speaks as an older brother, never condescending. He is NOT a chatbot — he is Taksha, an experienced carpentry trainee and Vocational certified trainer.

---

## 2. Target User

- **Who:** Low-literacy vegetable farmers in West Bengal and Bihar, age 18-45
- **Device:** Mid-range Android (4GB RAM, Chrome, 4G/5G)
- **Languages:** Bengali (primary), Hindi, English
- **Context:** Training for Vocational certification. Many are first-time smartphone users.
- **Key constraint:** Everything must work on slow connections. No unnecessary animations. Large tap targets. Voice-first where possible.

---

## 3. Core Experience — 6 Tabs

The app has a **sticky header** with two global dropdowns:
1. **Module Selector** — pick any of 38 modules. ALL tabs respond to this selection.
2. **Language Selector** — বাংলা / हिन्दी / English. Changes EVERYTHING globally (UI labels, content, chat language, quiz language).

And a **bottom tab bar** with 6 tabs:

### Tab 1: শেখো (Learn)
- Shows sections for the selected module (3-6 sections per module)
- Each section expands to show content (500-2000 words of real teaching material)
- Content is trilingual — fetched per language from database
- "Mark Complete" button per section
- Module header shows: icon, title, NOS code, theory hours, practical hours

### Tab 2: জিজ্ঞেস করো (Ask)
- Chat interface with Taksha (Claude Sonnet)
- Contextualized to the selected module ("user is studying Soil Nutrients")
- Voice input (mic button → speech recognition → send)
- Voice output (toggle: responses read aloud via Google TTS)
- Quick action buttons for common questions (5 per language)
- Plain text responses only (no markdown) — because responses convert to speech
- Chat history preserved per module across tab switches

### Tab 3: পরীক্ষা (Quiz)
- 5 MCQ questions generated by AI (Claude Haiku) per module
- 4 options (A/B/C/D), 1 correct answer
- Explanation shown after each answer
- Score tracked. 70% minimum to pass.
- Questions generated in the selected language

### Tab 4: ভিডিও (Video)
- Curated YouTube videos mapped to modules (21 videos total)
- Thumbnail preview → tap to play (iframe embed)
- Trilingual titles
- Filtered to selected module

### Tab 5: প্রয়োগ (Apply) — Phase 3
- Placeholder now ("Coming Soon")
- Future: AI-powered productivity self-assessment
- Record what you applied in the field → Claude evaluates
- Voice-first flow: record → transcribe → evaluate

### Tab 6: অগ্রগতি (Progress)
- Two views: Overall and Per-Module (toggle)
- Overall: modules completed, quizzes taken, avg score, badges earned
- Per-module: sections completed, quiz history
- 7 achievement badges (First Step, Soil Expert, Pest Shield, etc.)

## 5. AI Specifications

### Chat (Taksha persona)
- **Model:** Claude Sonnet (latest)
- **Max tokens:** 700
- **System prompt:** 252 lines covering all 9 compulsory NOS + 6 electives with specific measurements, procedures, crop names in 3 languages, pest/disease identification, IPM hierarchy, economics
- **Key rules:**
  - NEVER reveal AI identity
  - NO markdown (plain text only — for TTS conversion)
  - NO emoji
  - Match user's language (Bengali script if Bengali, Devanagari if Hindi)
  - 2-5 sentences for simple questions, 5-8 for teaching moments
  - Reference NOS/module codes when natural
  - Celebrate effort ("ভালো প্রশ্ন করেছো!")
  - If outside farming scope: "ভাই, আমি সবজি চাষের expert"

### Quiz Generation
- **Model:** Claude Haiku (latest)
- **Output:** JSON array of 5 MCQs
- **Each question:** text, 4 options, correct index (0-3), explanation
- **Language:** matches global language selector
- **Topic:** mapped from module ID to specific knowledge area

### Text-to-Speech
- **Service:** Google Cloud TTS
- **Voice:** bn-IN-Chirp3-HD-Orus (male, all languages)
- **Pre-processing:** strip markdown, fix numeric ranges ("2-3" → "2 to 3"), remove emoji, max 5000 chars

## 6. Data Model (10 Tables)

All tables prefixed `acharya_` (shared Supabase instance).

### Content Tables (admin-managed)
- **acharya_modules** — 38 rows. ID, NOS code, trilingual titles, icon, hours, sort_order, group_key
- **acharya_sections** — ~200 rows. FK to module, trilingual titles, sort_order, estimated_hours
- **acharya_content** — ~600 rows. FK to section, lang (bn/hi/en), markdown body, status (draft/review/published). One row per section per language.
- **acharya_videos** — 21 rows. YouTube ID, FK to module, trilingual titles, duration
- **acharya_badges** — 7 rows. ID, icon, trilingual titles, condition description

### Learner Tables
- **acharya_learners** — anonymous device-based identity. Optional name/phone. Preferred language.
- **acharya_progress** — per learner per module. Sections completed (JSON array), time spent, completed flag.
- **acharya_quiz_attempts** — score, total, cached questions as JSON, timestamp
- **acharya_earned_badges** — learner + badge + earned_at
- **acharya_apply_logs** — Phase 3 placeholder. Log type, JSON data.

### Key Design Decisions
- Content stored as **separate rows per language** (not JSON column) — allows per-language publish status
- Module metadata has all 3 language columns in same row (short strings, always loaded together)
- Content loaded **on-demand per section** (not all upfront)
- Progress: **localStorage first** (instant), Supabase sync (async, best-effort)

## 9. Content Depth (What "Chapter-Length" Means)

Each module section should be 500-2000 words of **real teaching content**. Not summaries. Example for M07 Section 3 (Organic Fertilizers):

> ভার্মিকম্পোস্ট তৈরি: কেঁচো (Eisenia fetida) + গোবর + জৈব বর্জ্য। গর্ত: ১m × ১m × ০.৩m (ছায়ায়)। নিচে ১০cm শুকনো পাতা/খড়, তার উপর ১৫cm গোবর + রান্নাঘরের বর্জ্য, ২০০-৩০০ কেঁচো ছাড়ো, পাটের বস্তা দিয়ে ঢাকো। প্রতি ১৫ দিনে উলটাও। ৪৫-৬০ দিনে তৈরি — কালো, দানাদার, মাটির গন্ধ। প্রয়োগ: ৫-১০ টন/হেক্টর...

This is the depth expected. Real procedures, real measurements, real Bengali field context.

## 14. Non-Negotiables

1. **Trilingual is not optional.** Every string, every content block, every quiz question must work in bn/hi/en.
2. **No markdown in chat responses.** They convert to speech. Plain flowing text only.
3. **Taksha never reveals he is AI.** He is an experienced carpentry trainee.
4. **Mobile-first.** If it doesn't work on a Rs 8000 Android phone on 4G, it doesn't ship.
5. **Content depth matters.** A 30-hour module cannot have 1 paragraph. It needs 3-6 sections of 500-2000 words each.
6. **All crop names in 3 languages.** ঢেঁড়স = भिन्डी = Okra. Always.
